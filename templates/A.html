Introduction

This is a website where you can check the nature of our any website. The name verifyURL is sel-explanatory and hence qualifies as the title of our website. Through this website we aim to detect phishing websites in real time! The quick advancements in the worldwide networking and communication technologies, has transferred our day by day life exercises on to cyberspace. This change has led to an open, mysterious and uncontrolled framework of the Internet which empowers an astounding stage for cyberattacks, which presents genuine security weaknesses for networks as well as for the standard computer clients in any event, even for the experienced ones. One among the cyberattacks is the phishing attacks, which is one of the common social engineering techniques used by phishers to gain sensitive information such as username, passwords, credit card or any other sensitive data from the end users.

Models

DNN-CNN model

A URL semantic-based phishing attack detection is performed using Deep Learning algorithms and Natural Language Processing techniques. A hybrid model is proposed which is constructed by combining the features of a DNN model and CNN model.Two types of features are passed into the model, which are hand-crafted features extracted from the URL and character level features. The features extracted from the URLs are based on the usual differences found between phishing and legitimate URLs. The models are trained using a dataset containing 100000 instances which are split in an 70:30 ratio for training and testing. The URL syntax based phishing attack detection is a simple and quick way of detecting phishing URLs as it classifies the URL based on the syntax of the URL. For performing such classifications, deep learning algorithms are the best as they are good at understanding complex features from the input data. In order to classify phishing URL using the syntax of the URL, we can train deep learning models in two ways, one by extracting the features from the URL, based on the common pattern or syntactical differences in the legitimate and phishing URLs and the second way is to encode these URLs and then train the model by passing the entire encoded URL. But as both of them are used for classifying the same target labels, it is more efficient when both can be done using the same model. So, the proposed model is designed in such a way that it can be trained by passing both the extracted features and the respective encoded URL as input. This model is built by using two input layers, one for the continuous data which are the values of the extracted features of shape (22,) and the other for the integer encoded URLs which are padded sequences with max length of 1000. The second input layer is followed by an embedding layer with embedded weights to convert the integer encodings of the URLs into one hot encodings. These encodings are then passed through convolution layer, max pooling layer, convolution layer and flatten layers in the sequential order respectively. By doing this, the convolution layers create feature maps, max pooling layer helps in creating pooled feature maps and the flatten layer reduces the size of the data from a matrix to column. This data is then passed through a dense fully connected layer. Parallelly, the first input layer is also passed through a dense fully connected layer. This is done so as to obtain the same shaped tensor outputs which can be passed as an input into the concatenation layers. The output of this layer is then passed through a set of six dense fully connected layers to obtain high accuracy and lastly it is passed through a dense sigmoid output layer for getting the classification results. This model is compiled using binary cross entropy and Adam as loss function and optimizer respectively.


The dataset used was made by our developer team by collecting lists of URLs through various resources present on the internet.
